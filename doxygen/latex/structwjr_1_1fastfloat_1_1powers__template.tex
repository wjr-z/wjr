struct wjr\+::fastfloat\+::powers\+\_\+template$<$ unused $>$}
When mapping numbers from decimal to binary, we go from w \texorpdfstring{$\ast$}{*} 10\texorpdfstring{$^\wedge$}{\string^}q to m \texorpdfstring{$\ast$}{*} 2\texorpdfstring{$^\wedge$}{\string^}p but we have 10\texorpdfstring{$^\wedge$}{\string^}q = 5\texorpdfstring{$^\wedge$}{\string^}q \texorpdfstring{$\ast$}{*} 2\texorpdfstring{$^\wedge$}{\string^}q, so effectively we are trying to match w \texorpdfstring{$\ast$}{*} 2\texorpdfstring{$^\wedge$}{\string^}q \texorpdfstring{$\ast$}{*} 5\texorpdfstring{$^\wedge$}{\string^}q to m \texorpdfstring{$\ast$}{*} 2\texorpdfstring{$^\wedge$}{\string^}p. 

Thus the powers of two are not a concern since they can be represented exactly using the binary notation, only the powers of five affect the binary significand. The smallest non-\/zero float (binary64) is 2\texorpdfstring{$^\wedge$}{\string^}−1074. We take as input numbers of the form w x 10\texorpdfstring{$^\wedge$}{\string^}q where w \texorpdfstring{$<$}{<} 2\texorpdfstring{$^\wedge$}{\string^}64. We have that w \texorpdfstring{$\ast$}{*} 10\texorpdfstring{$^\wedge$}{\string^}-\/343 \texorpdfstring{$<$}{<} 2\texorpdfstring{$^\wedge$}{\string^}(64-\/344) 5\texorpdfstring{$^\wedge$}{\string^}-\/343 \texorpdfstring{$<$}{<} 2\texorpdfstring{$^\wedge$}{\string^}-\/1076. However, we have that (2\texorpdfstring{$^\wedge$}{\string^}64-\/1) \texorpdfstring{$\ast$}{*} 10\texorpdfstring{$^\wedge$}{\string^}-\/342 = (2\texorpdfstring{$^\wedge$}{\string^}64-\/1) \texorpdfstring{$\ast$}{*} 2\texorpdfstring{$^\wedge$}{\string^}-\/342 \texorpdfstring{$\ast$}{*} 5\texorpdfstring{$^\wedge$}{\string^}-\/342 \texorpdfstring{$>$}{>} 2\texorpdfstring{$^\wedge$}{\string^}−1074. Thus it is possible for a number of the form w \texorpdfstring{$\ast$}{*} 10\texorpdfstring{$^\wedge$}{\string^}-\/342 where w is a 64-\/bit value to be a non-\/zero floating-\/point number.

Any number of form w \texorpdfstring{$\ast$}{*} 10\texorpdfstring{$^\wedge$}{\string^}309 where w\texorpdfstring{$>$}{>}= 1 is going to be infinite in binary64 so we never need to worry about powers of 5 greater than 308.
